Java Code Style Guidelines

1. Class Naming

  • Class names must use PascalCase (e.g., UserAccount, MainController).

2. Magic Numbers

  • Magic numbers are forbidden.
  • All numeric values with contextual meaning must be defined as named constants, declared as static final fields.
    Example: private static final int MAX_USERS = 100;

3. Boolean Method Naming

  • Methods that return a boolean value must begin with the prefix is (e.g., isValid(), isEmpty(), isAvailable()).

4. Procedure Naming (Void Methods)

  • Void methods must be named using verbs (e.g., printReport(), calculateSum()).
  • A procedure must do only what its name suggests (e.g., printResult() must only print, not calculate).

5. Local Variables

  • All local variables must have meaningful and descriptive names, using camelCase (e.g., totalScore, userName, inputBuffer).

6. Loop Variable Naming

  • For counters in simple loops, short names such as i, j, k, l are acceptable.

7. Spacing Around Operators

  • All binary operators must have exactly one space before and after.
  • ✅ a + b
  • ❌ a+b or ❌ a + b

8. Block Structures and Braces

  • All control structures (if, else, for, while, etc.) must use curly braces, even for single statements.
  • The opening brace { must be placed on the same line as the control statement.
if (isReady) {
    startProcessing();
}

9. Blank Lines

  • Unnecessary blank lines must be avoided.
  • Blank lines should be used only to separate logical sections of code and improve readability.

10. Indentation

  • Every block of code must be indented using a consistent, fixed-width indentation.
  • Recommended: 4 spaces per indentation level (no tabs).
